Overview

Welcome to the Coinley API documentation. This cryptocurrency payment gateway allows merchants to accept crypto payments across multiple blockchain networks with automated fee distribution via smart contracts.

Multi-Chain

Authentication

Coinley API uses two authentication methods depending on the user type:

Merchant API Authentication

For merchant endpoints, include your API Key and Secret in request headers: 

{
  "x-api-key": "your_api_key_here",
  "x-api-secret": "your_api_secret_here"
}

Important: Keep your API credentials secure. Never share them publicly or commit them to version control.

JWT Token Authentication

For admin and team member endpoints, use JWT tokens in the Authorization header: 

{
  "Authorization": "Bearer your_jwt_token_here"
}

Tokens are obtained from login endpoints and expire after 24 hours.

GET /api/payments/merchant/stats
Auth Required

Get comprehensive payment statistics and metrics for merchant dashboard. This endpoint provides real-time analytics including transaction counts, revenue totals, and fee calculations.

Dashboard Analytics

This endpoint powers the merchant dashboard with live transaction metrics, completion rates, revenue tracking, and average payment amounts. All amounts are returned as decimal strings for precision.

Headers:

x-api-key: your_api_key
x-api-secret: your_api_secret

Response (200 OK):

{
  "success": true,
  "stats": {
    "totalPayments": 1573,
    "completedPayments": 570,
    "pendingPayments": 801,
    "failedPayments": 202,
    "totalAmount": "245.34",
    "totalFeeAmount": "2.82",
    "averageAmount": "0.43"
  }
}

Response Fields:

totalPayments (integer)

Total number of PaymentSplitter transactions created by the merchant

completedPayments (integer)

Number of successfully completed transactions

pendingPayments (integer)

Number of payments awaiting confirmation on the blockchain

failedPayments (integer)

Number of failed or expired transactions

totalAmount (string - decimal)

Total revenue from completed payments (in base currency, e.g., USDT). Returned as string to preserve decimal precision.

totalFeeAmount (string - decimal)

Total fees collected from completed payments. Fees are automatically distributed via PaymentSplitter smart contracts.

averageAmount (string - decimal)

Average payment amount (totalAmount รท completedPayments). Useful for revenue analytics and forecasting.

Usage Tip: This endpoint is called by the merchant dashboard to display real-time statistics. The data is filtered to only include PaymentSplitter transactions for accurate fee tracking.

Example Usage (JavaScript):

// Fetch merchant stats for dashboard
const response = await fetch('https://api.coinley.io/api/payments/merchant/stats', {
  method: 'GET',
  headers: {
    'x-api-key': 'your_api_key',
    'x-api-secret': 'your_api_secret'
  }
});

const data = await response.json();

if (data.success) {
  console.log('Total Revenue:', data.stats.totalAmount);
  console.log('Completion Rate:',
    (data.stats.completedPayments / data.stats.totalPayments * 100).toFixed(1) + '%'
  );
  console.log('Average Transaction:', data.stats.averageAmount);
}

// Calculate dashboard metrics
const completionRate = (
  data.stats.completedPayments / data.stats.totalPayments * 100
).toFixed(1);

const dashboardStats = {
  totalRevenue: {
    value: data.stats.totalAmount,
    change: '+12.5%' // Calculate from historical data
  },
  totalTransactions: {
    value: data.stats.totalPayments,
    change: '+8.3%'
  },
  completedRate: {
    value: completionRate + '%',
    change: '+2.1%'
  },
  pendingTransactions: {
    value: data.stats.pendingPayments,
    change: '-5.2%'
  }
};

Error Codes

400 Bad Request

Invalid request parameters or validation error

{ "error": "Invalid amount format" }

401 Unauthorized

Missing or invalid authentication credentials

{ "error": "Authentication required - merchant info missing" }

403 Forbidden

Valid credentials but insufficient permissions

{ "error": "Invalid setup code" }

404 Not Found

Requested resource does not exist

{ "error": "Payment not found" }

500 Internal Server Error

Server encountered an unexpected error

{ "error": "Failed to create payment session", "message": "..." }

Error Response Format

All errors follow a consistent JSON structure:

{
  "error": "Brief error description",
  "message": "Detailed error message (optional)",
  "success": false
}
